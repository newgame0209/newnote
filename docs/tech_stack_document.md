# 技術スタックドキュメント

## 1. プロジェクト背景とモチベーション

### 1.1 事業概要

*   **プロジェクト名**: しゃべるノート
*   **コンセプト**: 手書きメモやテキストをAIの力で音声化し、ユーザーが何度も聞くことで記憶の定着をサポートするノートアプリ
*   **ターゲットユーザー**: 学生およびビジネスパーソン

## 2. 技術スタック

### 2.1 フロントエンド

*   **言語**: JavaScript, HTML, CSS
*   **ライブラリ/フレームワーク**: React.js
*   **UI/UXデザイン**: シンプルで白と薄い青色を基調にしたデザイン （参考: Notion）

### 2.2 バックエンド

*   **言語**: Python, JavaScript
*   **フレームワーク**: Flask (Python)またはExpress.js (Node.js)
*   **データベース**: SQLite または MongoDB（MVPフェーズでは簡易DBを想定）

### 2.3 AIとAPI

*   **音声変換AI**: 音声変換 API の選定中（OpenAI or Google TTS）
*   **OCR API**: 描画した手書き文字の識別にはPythonのOCRライブラリを検討中
*   **キャンバスライブラリ**: Fabric.js を利用

### 2.4 デプロイメント

*   **Webアプリ**: AWS または Herokuを利用してホスティングする計画

## 3. アーキテクチャ

### 3.1 アプリケーションフロー

1.  **ノート作成**

    *   ユーザーがノートを作成し、手書きで内容を入力
    *   手動で保存し、ノートはデータベースに保存

2.  **音声変換プロセス**

    *   保存された手書きデータをOCR APIでテキストに変換
    *   AI APIを利用しテキストを音声に変換し、ストリーミング再生可能

3.  **ユーザー体験**

    *   UI/UXは直感的に操作可能かつシンプル
    *   Apple Pencil や似たスタイラスの利用をサポート

## 4. セキュリティと非機能要件

### 4.1 MVPフェーズのセキュリティ

*   ユーザーログイン機能はMVPフェーズでは未実装
*   データ処理や保存では最低限のセキュリティを考慮

### 4.2 パフォーマンス要件

*   ノートごとのページ数の上限は10ページ
*   描画と音声変換の処理がユーザーの体験を阻害しないよう努力

## 5. 将来的な拡張

*   PDFや写真データを音声化する機能の追加
*   スマホおよびタブレットアプリへの展開

この技術スタックは現在のMVPフェーズにおける仮のものであり、ユーザーフィードバックや技術的要件に従って将来的に更新される可能性があります。
