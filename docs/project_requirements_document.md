# プロジェクト要件文書

## 1. プロジェクト概要

私たちのプロジェクトは、ノート作成用ウェブアプリケーション「しゃべるノート」の最小限の製品版 (MVP) を構築することにあります。このアイデアは「書いて覚える、聞いて定着する」というコンセプトに基づいており、主に学生やビジネスパーソンを対象に、学習の強化を支援します。ユーザーは手書きまたは入力されたノートを作成し、その後、AI 駆動の音声変換を使用してノートを聴くことができます。この方法は、視覚と聴覚の両方の学習チャネルを活用し、従来のノート作成方法に苦労している人々を支援します。

このプロジェクトは、手描きのノート入力と AI 駆動のテキスト読み上げ変換の統合の技術的実現可能性を探求し、検証することを目的としています。主な目標は、このアプローチの実行可能性を証明する機能的な MVP を提供することです。成功の定義は、直感的でシンプルなユーザーインターフェースを介したスムーズなノートの作成、編集、および音声への変換が可能であることです。この MVP は、将来的な拡張の基礎を築き（例：iOS/Android 版のモバイルアプリや、アップロードされた文書の追加音声変換機能）、モバイルアプリアップロードも目指します。

## 2. 範囲内と範囲外

**範囲内:**

*   ノートを作成、編集、保存できるウェブアプリケーション。
*   新しいノートの作成時に、用紙サイズ、方向、色、カテゴリの選択を可能にするセットアップモーダル。
*   ペン、消しゴム、マーカー、テキスト入力、画像アップロードを使用できる描画キャンバス。
*   ノートあたり最大10ページの対応のスワイプ型ページ移動。
*   メインカテゴリやサブカテゴリの選択とタイトルの割り当てを含む手動保存機能。
*   手書きや入力されたテキストを音声に変換する AI 駆動の音声変換機能。
*   音声タイプ（成人女性／成人男性）と言語設定（日本語のみ）を切り替え可能な設定画面。

**範囲外:**

*   ユーザー認証およびログイン機能。MVP にはログインシステムはありません。
*   アップロードされた PDF や写真の音声変換に関する高度な画像テキスト変換や OCR 機能。
*   モバイル（iOS/Android）ネイティブアプリ版の開発。最初の開発はウェブアプリに集中。
*   自動保存やオートセーブ機能、または記載されていない追加のページ操作ジェスチャー。
*   音声変換のカスタマイズの追加やモネタイズ向けのプレミアム機能。

## 3. ユーザーフロー

典型的なユーザーのジャーニーは、ホーム画面から始まり、既存のノートの一覧を表示します。メインのインターフェースには、ノートを日付、カテゴリ、またはタイプでソートするオプションがあり、ユーザーはグリッドとリストビューの切り替えが可能です。ユーザーが「新しいノート」ボタンをクリックすると、モーダルウィンドウが開きます。このモーダルで、ユーザーは用紙サイズ（A4、A3、A7）、方向（縦または横）、色（白または黄）を選択し、メインおよびサブカテゴリを設定し、タイトルを提供します。

ノートが作成されると、アプリはノート編集画面に遷移します。ここで、ユーザーは指や Apple Pencil の入力に対応した固定の描画キャンバス上で作業します。ツールバーには、ペン、消しゴム、マーカー、テキスト入力、画像アップロードなどの描画ツールが用意されています。ユーザーは、創造的な入力を行った後、AI 音声変換ボタンをタップし、手書きまたは入力されたテキストを外部のテキスト読み上げ API に送信してオーディオを生成して再生します。ノートは手動で保存され、再開時には、アプリが最後に編集されたページを復元して継続性を確保します。

## 4. コア機能

*   **ノート作成と編集:**

    *   モーダルベースの新しいノートセットアップ（用紙サイズ、方向、色、カテゴリとタイトルを選択）。
    *   ペン、消しゴム、マーカー、テキスト入力、画像アップロードのツールを備えた描画キャンバス。
    *   最大10ページのノートに対する対応、スワイプでのページ移動が可能。
    *   再開時に最後に編集されたページを復元。
    *   モバイル／タブレット版では、ズームやスクロールを無効にしてキャンバスの安定性を確保。

*   **AI駆動の音声変換:**

    *   手書きおよび入力されたテキストを音声に変換する AI TTS API の利用。
    *   ノート編集画面内で直接変換された音声のストリーミング再生。
    *   制限：手書きテキストと入力されたテキストのみが変換対象（画像は除外）。

*   **設定画面:**

    *   異なる音声タイプ（例：成人女性、成人男性）への切り替えが可能。
    *   言語設定は現在日本語に限定。

*   **ナビゲーションとユーザーインターフェース:**

    *   日付、カテゴリ別にソート可能なノートリストビューがあるホーム画面。
    *   ノートの作成、編集、削除、設定オプションへの簡単なアクセス。
    *   シンプルで、Notion を感じさせる白とライトブルーのカラースキームによるクリーンなデザイン。

## 5. 技術スタックとツール

*   **フロントエンド:**

    *   核心的なウェブアプリケーションのための JavaScript、HTML、および CSS。（Tailwind CSS、shadcn/ui、React）
    *   Handwriting の入力を処理するための Fabric.js。

*   **バックエンド:**

    *   バックエンドロジックのための Python (将来のイテレーションで OCR 機能を統合する場合は特に)。
    *   AI TTS の統合には、OpenAI および Google TTS の API を使用（技術評価に基づいて変更される可能性あり）。

*   **追加ツール:**

    *   開発のための AI コーディング機能を備えた現代 IDE Windurfs。
    *   TTS API コールでトリガーされたオーディオ再生のためのストリーミング機能の統合。

## 6. 非機能要件

*   **パフォーマンス:**

    *   アプリケーションは、ノートあたり最大10ページをサポートし、顕著なパフォーマンスの低下はなくする必要があります。
    *   特に AI 音声変換リクエスト中のレスポンシブなページ移行と最小限のレイテンシーを確保します。

*   **セキュリティ:**

    *   MVP にログインや認証システムはありませんが、すべてのデータ伝送（例：TTS 用の API コール）は安全なチャネルを使用する必要があります。

*   **使いやすさ:**

    *   インターフェースは直感的でシンプルであり、ユーザーがノートの作成、編集、設定を簡単にナビゲートできるようにする必要があります。
    *   デザインは Notion にインスパイアされ、ミニマリストの白とライトブルーのカラーパレットを利用して視覚的に魅力的である必要があります。

*   **コンプライアンス:**

    *   すべてのサードパーティ製 API 統合が各々の利用規約を遵守する必要があります。

## 7. 制約と仮定

*   MVP は完全に Web ベースの展開に専念しており、将来的にはネイティブ iOS および Android アプリへの展開の可能性があります。
*   AI 音声変換の API コールは、（手書き認識後の）テキストから音声変換までの複数の連続リクエストを含む可能性があり、パフォーマンスの関心を引き起こす可能性があります。
*   ユーザーは Apple Pencil とタッチ入力の両方を使用すると想定されており、描画機能は両方を正確に検出し処理する必要があります。
*   手書きおよび入力されたテキストのみがオーディオ変換の対象であり、画像は明示的に除外されています。
*   設計と性能の考慮は、MVP 用に定義されたページの限られた数（ノートあたり10ページ）に基づいています。

## 8. 既知の問題と潜在的な落とし穴

*   **API コールによる負荷の高い処理:**

    *   ワークフローは、手書き入力をテキストに変換し（必要ならば）、その後テキスト読み上げ変換を実行するための複数の API コールを伴います。これにより、処理の負荷が高くなり、遅延やラグが生じる可能性があります。
    *   これを緩和するために、キャッシュ戦略やスロットルメカニズムを実装して API レート制限を管理することを検討してください。

*   **統合の複雑さ:**

    *   クライアントサイド描画（Fabric.js の使用）とバックエンドサービス（OCR/TTS を処理する Python）の間の統合は、同期化およびデータ処理の課題を導入する可能性があります。
    *   明確な API インターフェースの仕様と堅牢なエラーハンドリングを早期に確立する必要があります。

*   **ユーザー体験:**

    *   モバイルデバイスでのズームおよびスクロールの防止は描画能力を円滑に提供しながら難しいかもしれません。
    *   実際のデバイスを使用して継続的にテストし、フィンガーと Apple Pencil の両方の入力を使用して一貫したユーザーエクスペリエンスを確保することが重要です。

*   **将来のスケーラビリティ:**

    *   MVP では、PDF アップロード、高度な OCR 画像サポート、広範なモバイルサポートなどの高度な機能を除外しますが、これらの将来的な拡張は、主要なリファクタリングを避けるために事前に計画する必要があります。
    *   将来の反復での拡張を容易にするために、ドキュメントとコードの構造をきれいに維持する必要があります。

このドキュメントは「しゃべるノート」MVPの開発に関する単一の真実の源を提供することを意図しています。すべての後続の技術文書および実装計画はこれらの仕様に一致する必要があります。
